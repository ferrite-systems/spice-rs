# Modified Nodal Analysis

Every SPICE simulator works the same way: it turns a circuit into a matrix equation ${tex`Gx = b`}, then solves for ${tex`x`}. The method for building that matrix is called **Modified Nodal Analysis** (MNA).

This chapter shows exactly how MNA works — how each component becomes entries in a matrix, and how solving that matrix gives you every voltage and current in the circuit.

```js
import {kdlToSvg, SimBuilder} from "../components/spice.js";
import {Resistance, Voltage, formatSpice, formatEE} from "../components/ee-inputs.js";
import {simPanel} from "../components/readout.js";
```

## The unknowns

The solution vector ${tex`x`} contains two kinds of unknowns:

1. **Node voltages** — one for every node in the circuit (except ground, which is defined as 0V)
2. **Branch currents** — one for every voltage source and every inductor

If a circuit has ${tex`n`} nodes (excluding ground) and ${tex`m`} voltage sources, the matrix is ${tex`(n+m) \times (n+m)`}.

> The "Modified" in MNA refers to the addition of branch currents. Plain Nodal Analysis only solves for node voltages, which makes it unable to handle voltage sources directly. MNA adds extra equations for each voltage source, making the system slightly larger but much more general.

---

## Node voltages

The solution vector in MNA starts with one unknown per circuit node — except ground. Understanding why ground is special, and how nodes get numbered, clarifies the entire matrix structure.

### Ground is the reference, not a variable

Voltage is always measured *between* two points. To get absolute numbers, SPICE defines one node as the reference and fixes it at 0V. This is ground — node 0 in SPICE netlists.

Ground does not appear in the solution vector. It has no row and no column in the matrix. Every other node voltage is implicitly measured with respect to it. When we write ${tex`V_a = 5\text{V}`}, we mean the potential difference between node ${tex`a`} and ground is 5V.

This is why every SPICE netlist must have a node 0. Without a reference, the system of equations is underdetermined — you can add any constant to all voltages and still satisfy KCL. Fixing ground removes that degree of freedom and makes the solution unique.

### Node numbering

In a netlist, nodes have names: `in`, `out`, `vdd`, `0`. Internally, the simulator assigns each non-ground node a sequential index starting at 1. This index is the row (and column) position in the MNA matrix.

### Internal vs external nodes

Some devices create nodes that don't appear in the netlist. A MOSFET model, for example, adds internal nodes for the parasitic drain and source resistances. These nodes are real — they get indices, rows, and columns — but the user never names them.

In spice-rs (following ngspice), external nodes come first in the numbering, internal nodes are appended after. A circuit with 5 external nodes and 3 internal nodes has 8 node voltage unknowns, occupying rows 1 through 8 of the solution vector.

### The solution vector

Putting it together, the full solution vector ${tex`x`} for a circuit with ${tex`n`} nodes and ${tex`m`} voltage sources is:

```tex
x = \begin{bmatrix} V_1 \\ V_2 \\ \vdots \\ V_n \\ I_{V1} \\ \vdots \\ I_{Vm} \end{bmatrix}
```

The first ${tex`n`} entries are node voltages — external then internal. The remaining ${tex`m`} entries are branch currents for voltage sources and inductors. The matrix is ${tex`(n+m) \times (n+m)`}, and a single solve gives every voltage and current in the circuit.

## Conductance stamps

Each component contributes entries to the matrix ${tex`G`} and the right-hand side ${tex`b`}. These contributions are called **stamps** — the component "stamps" its values into the matrix.

The resistor stamp is the atom of circuit simulation. Every other stamp — capacitors, diodes, transistors — is a variation on this pattern. Understand it once, and the rest follows.

### Resistor

A resistor ${tex`R`} between nodes ${tex`i`} and ${tex`j`} has conductance ${tex`G = 1/R`}. It carries current ${tex`I = G(V_i - V_j)`}. Write KCL at both nodes:

**Node ${tex`i`}:** current leaves through the resistor, so ${tex`-G(V_i - V_j)`} is the resistor's contribution. Expanding: ${tex`-GV_i + GV_j`}.

**Node ${tex`j`}:** current enters through the resistor, so ${tex`+G(V_i - V_j)`}. Expanding: ${tex`+GV_i - GV_j`}.

These coefficients go into a 2x2 stamp pattern:

```tex
G[i,i] \mathrel{+}= G \qquad G[i,j] \mathrel{-}= G
```
```tex
G[j,i] \mathrel{-}= G \qquad G[j,j] \mathrel{+}= G
```

Positive on the diagonal, negative on the off-diagonal. Symmetric. This is the most important stamp in SPICE — it shows up everywhere because even nonlinear devices are linearized into an equivalent conductance at each iteration.

Here is how spice-rs implements it:

```rust
// From device/resistor.rs — the load() function
let g = self.conductance;
mna.stamp(self.pos_node, self.pos_node,  g);
mna.stamp(self.neg_node, self.neg_node,  g);
mna.stamp(self.pos_node, self.neg_node, -g);
mna.stamp(self.neg_node, self.pos_node, -g);
```

Four calls to `stamp()`. Each one adds a value to one matrix entry. That's all a resistor does.

### Stamping two resistors

Take two resistors: ${tex`R_1`} between nodes 1 and 2 (${tex`G_1 = 0.001`} S), and ${tex`R_2`} between nodes 2 and ground (${tex`G_2 = 0.002`} S).

${tex`R_1`} stamps its 2x2 block. ${tex`R_2`} connects node 2 to ground — ground has no row or column, so only the diagonal entry at node 2 survives. The final matrix is the sum of all stamps:

| | ${tex`V_1`} | ${tex`V_2`} |
|---|---|---|
| **Row 1** | ${tex`0.001`} | ${tex`-0.001`} |
| **Row 2** | ${tex`-0.001`} | ${tex`0.003`} |

This is the key insight: each component stamps independently, and the stamps superpose by addition. The order doesn't matter. The components don't know about each other. The matrix assembles itself.

### When one node is ground

When a resistor connects to ground (node 0), that node has no row or column. The 2x2 stamp degenerates: the row and column for ground are simply discarded. Only the diagonal entry at the other node survives. This is not a special case in the code — ground is just absent from the matrix, so stamps that reference it naturally contribute only their non-ground entries.

### Current source

An independent current source ${tex`I`} from node ${tex`j`} to node ${tex`i`} (current flows from ${tex`j`} to ${tex`i`}) stamps only the right-hand side:

```tex
b[i] \mathrel{+}= I \qquad b[j] \mathrel{-}= I
```

No matrix entries — a current source doesn't depend on any node voltage, so it contributes no conductance.

### Voltage source

A voltage source ${tex`V_s`} between nodes ${tex`i`} and ${tex`j`} (positive terminal at ${tex`i`}) adds a new unknown: the branch current ${tex`I_{Vs}`}. If this is the ${tex`k`}-th branch variable, it stamps:

```tex
G[i, n{+}k] \mathrel{+}= 1 \qquad G[j, n{+}k] \mathrel{-}= 1
```
```tex
G[n{+}k, i] \mathrel{+}= 1 \qquad G[n{+}k, j] \mathrel{-}= 1
```
```tex
b[n{+}k] \mathrel{+}= V_s
```

The first two rows express KCL: the branch current ${tex`I_{Vs}`} enters node ${tex`i`} and leaves node ${tex`j`}. The last row enforces the voltage constraint: ${tex`V_i - V_j = V_s`}.

This is why voltage sources add a row and column to the matrix — they introduce both a new unknown (the branch current) and a new equation (the voltage constraint).

### Why the current comes out negative

When you simulate a simple circuit with a voltage source supplying power, the branch current ${tex`I_{Vs}`} is typically negative. This is not a bug — it's a consequence of the sign convention. MNA defines ${tex`I_{Vs}`} as current flowing from the positive terminal through the source to the negative terminal (i.e., internally). In a source supplying current to the circuit, current flows out of the positive terminal externally, which is the opposite direction. Hence the negative sign.

## Building the full matrix

Let's assemble a complete MNA matrix from scratch. Every stamp, every entry, laid out so you can trace each number back to the component that put it there.

The voltage divider from Chapter 1:

**Nodes:** `in` = 1, `mid` = 2, ground = 0 (excluded). **Branch variables:** ${tex`I_{V1}`} at position 3. Matrix size: ${tex`3 \times 3`}.

**Stamp R1** (1k between nodes 1 and 2): ${tex`G_1 = 1/1000 = 0.001`} S. Adds ${tex`+G_1`} to (1,1) and (2,2), ${tex`-G_1`} to (1,2) and (2,1).

**Stamp R2** (1k between node 2 and ground): ${tex`G_2 = 0.001`} S. Ground has no row/column, so only the (2,2) diagonal entry survives. Node 2 diagonal is now ${tex`G_1 + G_2 = 0.002`}.

**Stamp V1** (10V source, positive at node 1, negative at ground): Branch current ${tex`I_{V1}`} at index 3. Adds 1's coupling the current into KCL at node 1, and the constraint equation in row 3.

The complete system:

```tex
\begin{bmatrix} 0.001 & -0.001 & 1 \\ -0.001 & 0.002 & 0 \\ 1 & 0 & 0 \end{bmatrix} \begin{bmatrix} V_{in} \\ V_{mid} \\ I_{V1} \end{bmatrix} = \begin{bmatrix} 0 \\ 0 \\ 10 \end{bmatrix}
```

Each row tells a story:

- **Row 1** (KCL at `in`): ${tex`0.001 V_{in} - 0.001 V_{mid} + I_{V1} = 0`}. The current through R1 plus the current from V1 sum to zero.
- **Row 2** (KCL at `mid`): ${tex`-0.001 V_{in} + 0.002 V_{mid} = 0`}. Current into `mid` from R1 equals current out through R2.
- **Row 3** (V1 constraint): ${tex`V_{in} = 10`}. The source forces the voltage.

Solving gives ${tex`V_{in} = 10\text{V}`}, ${tex`V_{mid} = 5\text{V}`}, ${tex`I_{V1} = -0.01\text{A}`}. The voltage divider divides, and the source supplies 10 mA.

## Solving the system

The matrix is built. Now what? SPICE needs to solve ${tex`Gx = b`} — and it needs to do it fast, because nonlinear circuits require solving this system dozens of times per operating point.

### LU factorization

The workhorse is LU factorization: decompose ${tex`G`} into a lower-triangular matrix ${tex`L`} and an upper-triangular matrix ${tex`U`} such that ${tex`G = LU`}. Then solving ${tex`Gx = b`} becomes two easy steps:

1. **Forward substitution:** solve ${tex`Ly = b`} for ${tex`y`} (top to bottom, each equation has one new unknown)
2. **Back substitution:** solve ${tex`Ux = y`} for ${tex`x`} (bottom to top, same idea)

For a dense ${tex`n \times n`} matrix, LU factorization is ${tex`O(n^3)`}. But circuit matrices are not dense — and that changes everything.

### Sparsity

A 1000-node circuit has a ${tex`1000 \times 1000`} conductance matrix — one million entries. But each component touches only 2 to 4 nodes, so the vast majority of entries are zero. A typical circuit matrix is 99% zeros.

The sparse solver stores only the nonzero entries and operates only on those. Instead of ${tex`O(n^3)`}, the factorization runs in time roughly proportional to the number of nonzero entries — effectively ${tex`O(n)`} for typical circuits. This is what makes SPICE practical for large designs.

spice-rs uses a Markowitz-ordered LU factorization, faithfully ported from the KLU algorithm in SuiteSparse. Chapter 19 covers the sparse solver in full detail — the pivot ordering strategies, fill-in minimization, and the data structures that make it efficient.

### From solution to results

After the solve, the solution vector ${tex`x`} contains everything:

**Node voltages** occupy the first ${tex`n`} positions. These are the primary output — the voltage at every node in the circuit, measured with respect to ground.

**Branch currents** for voltage sources and inductors occupy positions ${tex`n+1`} through ${tex`n+m`}. These fall out of the solve for free — no extra computation needed.

**Other currents** (through resistors, into transistor terminals) are computed after the fact from the node voltages. For a resistor: ${tex`I = G(V_i - V_j)`}. For a MOSFET: evaluate the device equations at the solved terminal voltages.

### What happens next

For a linear `.OP` analysis, the solve is done — one factorization, one forward/back substitution, and the answer is ready. But for nonlinear circuits, this is just one iteration of Newton-Raphson:

1. Linearize all devices at the current operating point
2. Stamp the linearized values into the matrix
3. Solve
4. Update the operating point
5. Check convergence — if not converged, go to step 1

The matrix is re-stamped and re-factored at every iteration. Factorization dominates the runtime, which is why the sparse solver matters so much — it's the innermost loop of the entire simulation.

---

## The stamp pattern

Every component in SPICE reduces to the same operation: *stamp values into the matrix and RHS*. This is true for resistors, capacitors, diodes, MOSFETs, transmission lines — everything.

For linear components (resistors, linear capacitors at a given frequency), the stamps are constant. For nonlinear components (diodes, transistors), the stamps change at each Newton-Raphson iteration as the linearization point updates. But the mechanism is identical: call `mna.stamp()` with a row, column, and value.

This uniformity is the key insight of MNA — it reduces the entire problem of circuit simulation to: *fill a matrix, solve it, repeat*.

The next chapter shows what happens when the stamps aren't constant — when Newton-Raphson iteration is needed to handle nonlinear devices.
