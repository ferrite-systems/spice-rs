# AC Analysis

```js
import {Resistance, Capacitance, Inductance, Voltage, formatSpice, formatEE} from "../components/ee-inputs.js";
```

AC analysis answers the question: *how does this circuit respond to signals at different frequencies?*

Every amplifier has a bandwidth. Every filter has a rolloff. Every feedback loop has a phase margin. AC analysis is how you see these things -- it sweeps across frequency, from hertz to gigahertz, and tells you the gain and phase shift at every point along the way.

The key insight is that AC analysis doesn't simulate a time-domain signal. It never applies a sine wave and watches the output. Instead, it takes a shortcut: *linearize everything at the DC operating point, then solve the resulting linear system at each frequency.* Because the linearized circuit is a purely linear problem, there's no Newton-Raphson iteration, no convergence concerns, no timestep control. Just one matrix solve per frequency point -- and each solve is complex-valued, because impedances at nonzero frequency have both magnitude and phase.

This is what makes AC analysis fast. A transient simulation of a 10 MHz sine wave through an amplifier might need thousands of timesteps. AC analysis gets the same frequency response information by solving a few hundred linear systems.

The cost: **the signal must be small enough that the linearization is valid.** AC analysis cannot tell you about clipping, distortion, or any behavior that depends on signal amplitude. For large signals, you need transient analysis (Chapter 9).

---

## Small-Signal Linearization

AC analysis only works because of a powerful simplification: at the DC operating point, every nonlinear device can be replaced by a *linear* small-signal model.

The DC operating point gives each device a specific set of voltages and currents. These define a single point on the device's nonlinear I-V curve. If we add a *tiny* AC signal -- small enough that we don't move far from that point -- the device's response is approximately linear.

### The diode: one conductance, one capacitance

The small-signal conductance is:

```tex
g_d = \frac{dI_D}{dV_D} = \frac{I_D + I_S}{V_T} \approx \frac{I_D}{V_T}
```

For a diode biased at ${tex`I_D = 1`} mA: ${tex`g_d \approx 38`} mS, or a small-signal resistance of about ${tex`26\ \Omega`}.

Together with a diffusion capacitance ${tex`C_d`}, these are all that AC analysis needs from the diode. The exponential equation is gone. The device is fully linear.

### The MOSFET: a richer model

The core small-signal parameters:

- **${tex`g_m`} (transconductance)** -- how much drain current changes when ${tex`V_{GS}`} changes. This is the gain mechanism.
- **${tex`g_{ds}`} (output conductance)** -- how much drain current changes when ${tex`V_{DS}`} changes. Channel-length modulation gives a small nonzero ${tex`g_{ds}`}.
- **${tex`g_{mbs}`} (body transconductance)** -- how ${tex`V_{BS}`} modulates drain current via the body effect.
- **Capacitances** -- ${tex`C_{gs}`}, ${tex`C_{gd}`}, ${tex`C_{gb}`}, ${tex`C_{bs}`}, ${tex`C_{bd}`}. These determine the high-frequency behavior: as frequency increases, current flows through the capacitances and gain rolls off.

### The key assumption

All of this rests on one assumption: **the AC signal is infinitesimally small.** The linearization is only valid for perturbations small enough that the nonlinear terms are negligible. This is why AC analysis has no concept of signal amplitude -- the circuit is perfectly linear in the AC world.

## Complex Impedance and the AC Matrix

In DC analysis, the MNA matrix is real-valued. In AC analysis, the matrix becomes *complex*. This is where frequency enters the picture.

### Why complex numbers?

A capacitor's behavior depends on frequency. The current through a capacitor leads the voltage by 90 degrees. Complex numbers encode both magnitude and phase:

```tex
Z_C = \frac{1}{j\omega C}
```

Similarly, an inductor: ${tex`Z_L = j\omega L`}. A resistor remains ${tex`Z_R = R`}: purely real, no phase shift.

### The AC MNA matrix

At each frequency point, the system to solve is:

```tex
(G + j\omega C)\mathbf{x} = \mathbf{b}
```

The real part ${tex`G`} contains conductances from resistors and small-signal parameters. The imaginary part ${tex`j\omega C`} contains admittances from capacitors and small-signal capacitances.

**Resistor** stamps ${tex`g = 1/R`} into the real part. **Capacitor** stamps ${tex`j\omega C`} into the imaginary part -- at low frequency it vanishes (open circuit), at high frequency it dominates (short circuit).

### The frequency sweep

For each frequency point, spice-rs computes ${tex`\omega = 2\pi f`}, assembles the complex matrix by calling `ac_load()` on every device, factors and solves the complex system, and stores the complex node voltages. The solution at each point gives magnitude (gain) and phase.

## Bode Plots

The result of AC analysis is a table: for each frequency, a complex voltage at every node. The Bode plot is the standard visualization -- two plots showing magnitude and phase vs. frequency.

### Magnitude: decibels

```tex
|H(f)|_{\text{dB}} = 20 \log_{10} |H(f)|
```

| Gain (linear) | Gain (dB) | Meaning |
|:-:|:-:|:--|
| 1 | 0 dB | Unity |
| 0.707 | -3 dB | Half-power point (cutoff frequency) |
| 0.1 | -20 dB | One-tenth |
| 10 | +20 dB | Ten times |

### Phase: degrees

```tex
\angle H(f) = \arctan\!\left(\frac{\text{Im}(H(f))}{\text{Re}(H(f))}\right)
```

A phase of ${tex`-180^\circ`} means the output is inverted -- in a feedback system, this is where oscillation can occur.

### Frequency sweep types

**DEC (decade)** -- logarithmically spaced, most common. Each step multiplies by a constant factor.

**OCT (octave)** -- logarithmically spaced per octave, common in audio.

**LIN (linear)** -- uniformly spaced, useful for narrow-range zoom.

### What Bode plots tell you

- **Bandwidth** -- the frequency range within 3 dB of the peak
- **Rolloff rate** -- first-order filter: -20 dB/decade; second-order: -40 dB/decade
- **Resonance** -- a peak above the passband level
- **Phase margin** -- stability indicator for feedback amplifiers

## Filter Circuits

### RC Low-Pass Filter

The simplest filter: one resistor, one capacitor. Low frequencies pass through; high frequencies are attenuated.

```js
const rcR = view(Resistance({label: "R", value: 1000}));
const rcC = view(Capacitance({label: "C", value: 100e-9}));
```

The transfer function is:

```tex
H(f) = \frac{1}{1 + j\omega RC}
```

The magnitude and phase:

```tex
|H(f)| = \frac{1}{\sqrt{1 + (f/f_c)^2}}, \qquad \angle H(f) = -\arctan\!\left(\frac{f}{f_c}\right)
```

where the cutoff frequency is:

```tex
f_c = \frac{1}{2\pi RC}
```

```js
const fc = 1 / (2 * Math.PI * rcR * rcC);
display(html`<p style="font-size:1.1em;">${tex`f_c = \frac{1}{2\pi \cdot ${formatEE(rcR, "Ω")} \cdot ${formatEE(rcC, "F")}}`} = <strong>${formatEE(fc, "Hz")}</strong></p>`);
```

```js
{
  const freqResponse = [];
  const fMin = fc / 100;
  const fMax = fc * 100;
  for (let i = 0; i <= 500; i++) {
    const f = fMin * Math.pow(fMax / fMin, i / 500);
    const w = 2 * Math.PI * f;
    const mag = 1 / Math.sqrt(1 + Math.pow(w * rcR * rcC, 2));
    const mag_db = 20 * Math.log10(mag);
    const phase_deg = -Math.atan(w * rcR * rcC) * 180 / Math.PI;
    freqResponse.push({f, mag_db, phase_deg});
  }
  display(Plot.plot({
    width: 640,
    height: 300,
    x: {type: "log", label: "Frequency (Hz)"},
    y: {label: "Magnitude (dB)", grid: true, domain: [-60, 5]},
    marks: [
      Plot.line(freqResponse, {x: "f", y: "mag_db", stroke: "#b87333", strokeWidth: 2}),
      Plot.ruleY([-3], {stroke: "#4a6fa5", strokeDasharray: "4"}),
      Plot.ruleX([fc], {stroke: "#4a6fa5", strokeDasharray: "4"}),
      Plot.text([{x: fc, y: -3}], {x: "x", y: "y", text: d => `-3 dB at ${formatEE(fc, "Hz")}`, dx: 60, dy: -10, fill: "#4a6fa5"})
    ]
  }));
  display(Plot.plot({
    width: 640,
    height: 200,
    x: {type: "log", label: "Frequency (Hz)"},
    y: {label: "Phase (deg)", grid: true, domain: [-100, 10]},
    marks: [
      Plot.line(freqResponse, {x: "f", y: "phase_deg", stroke: "#b87333", strokeWidth: 2}),
      Plot.ruleY([-45], {stroke: "#4a6fa5", strokeDasharray: "4"}),
      Plot.ruleX([fc], {stroke: "#4a6fa5", strokeDasharray: "4"})
    ]
  }));
}
```

At the cutoff frequency: magnitude = -3.01 dB, phase = -45 degrees. Below ${tex`f_c`}: signal passes through unchanged. Above ${tex`f_c`}: gain drops at -20 dB/decade.

### SPICE AC Analysis

The plot above was computed from the closed-form transfer function -- no simulator involved. Now let's run the **real SPICE engine** on the same RC filter and see what AC analysis produces.

```js
import {SimBuilder} from "../components/spice.js";
```

```js
{
  const filterKdl = `circuit "RC Filter" {
      group "rc" {
          component "R1" type="resistor" { value "${formatSpice(rcR)}"; port "1" net="vin"; port "2" net="vout"; place col=0 row=0 }
          component "C1" type="capacitor" { value "${formatSpice(rcC)}"; port "1" net="vout"; port "2" net="gnd"; place col=0 row=2 }
      }
      node "gnd" role="ground"
      node "vout" label="Vout"
  }`;

  const acResult = await SimBuilder.fromKdl(filterKdl)
    .acSource("VIN", {dc: 0, mag: 1, phase: 0})
    .acSweep({sweep: "dec", npts: 20, fstart: 1, fstop: 1e7})
    .measure("Vout", "voltage", "vout")
    .run();

  const magData = acResult.frequencies.map((f, i) => ({
    f,
    mag_db: 20 * Math.log10(acResult.signals_mag["v(vout)"][i]),
    phase_deg: acResult.signals_phase["v(vout)"][i]
  }));

  // Find the -3dB point by interpolation
  let f3db = fc;
  for (let i = 1; i < magData.length; i++) {
    if (magData[i].mag_db <= -3 && magData[i - 1].mag_db > -3) {
      const t = (-3 - magData[i - 1].mag_db) / (magData[i].mag_db - magData[i - 1].mag_db);
      f3db = magData[i - 1].f * Math.pow(magData[i].f / magData[i - 1].f, t);
      break;
    }
  }

  // Magnitude plot
  display(Plot.plot({
    width: 640,
    height: 300,
    x: {type: "log", label: "Frequency (Hz)"},
    y: {label: "Magnitude (dB)", grid: true, domain: [-60, 5]},
    marks: [
      Plot.line(magData, {x: "f", y: "mag_db", stroke: "#b87333", strokeWidth: 2}),
      Plot.ruleY([-3], {stroke: "#4a6fa5", strokeDasharray: "4"}),
      Plot.ruleX([fc], {stroke: "#4a6fa5", strokeDasharray: "4"}),
      Plot.dot([{x: f3db, y: -3}], {x: "x", y: "y", r: 5, fill: "#4a6fa5"}),
      Plot.text([{x: f3db, y: -3}], {
        x: "x", y: "y",
        text: d => `-3 dB at ${formatEE(f3db, "Hz")}`,
        dx: 80, dy: -10, fill: "#4a6fa5", fontSize: 12
      })
    ]
  }));

  // Phase plot
  display(Plot.plot({
    width: 640,
    height: 200,
    x: {type: "log", label: "Frequency (Hz)"},
    y: {label: "Phase (deg)", grid: true, domain: [-100, 10]},
    marks: [
      Plot.line(magData, {x: "f", y: "phase_deg", stroke: "#b87333", strokeWidth: 2}),
      Plot.ruleY([-45], {stroke: "#4a6fa5", strokeDasharray: "4"}),
      Plot.ruleX([fc], {stroke: "#4a6fa5", strokeDasharray: "4"}),
      Plot.dot([{x: fc, y: -45}], {x: "x", y: "y", r: 5, fill: "#4a6fa5"}),
      Plot.text([{x: fc, y: -45}], {
        x: "x", y: "y",
        text: d => `-45° at fc`,
        dx: 50, dy: -10, fill: "#4a6fa5", fontSize: 12
      })
    ]
  }));

  display(html`<p style="font-size:0.85em;color:#6b5d4d;font-style:italic">
    SPICE AC sweep: ${acResult.frequencies.length} frequency points from 1 Hz to 10 MHz (20 points per decade).
    The -3 dB point falls at ${formatEE(f3db, "Hz")} — the analytical cutoff is ${formatEE(fc, "Hz")}.
  </p>`);
}
```

The SPICE result matches the analytical formula exactly -- as it should for a linear circuit. The simulator assembles the complex MNA matrix ${tex`(G + j\omega C)\mathbf{x} = \mathbf{b}`} at each frequency point and solves it directly. No Newton-Raphson needed, no timestep control, just one complex matrix factorization per frequency. The blue dot marks the -3 dB point where magnitude crosses ${tex`-3.01`} dB and phase passes through ${tex`-45°`}.

The value of the SPICE approach becomes clear when you move beyond passive filters to circuits with transistors, where no closed-form transfer function exists. The simulator linearizes each device at the DC operating point and sweeps -- the same machinery, applied to arbitrarily complex circuits.

### RLC Series Resonance

Add an inductor and the circuit can *resonate*. At one specific frequency, the inductor and capacitor impedances cancel, leaving only the resistance.

The resonant frequency:

```tex
f_0 = \frac{1}{2\pi\sqrt{LC}}
```

The quality factor:

```tex
Q = \frac{1}{R}\sqrt{\frac{L}{C}} = \frac{f_0}{\Delta f_{-3\text{dB}}}
```

A high-${tex`Q`} circuit has a narrow, tall resonance peak. A low-${tex`Q`} circuit has a broad, gentle peak.

### From filters to amplifiers

These passive filter examples demonstrate the mechanics of AC analysis without nonlinear device complexity. But the real power of AC analysis is in circuits with transistors, where the small-signal linearization comes into play. An amplifier's frequency response is shaped by the same physics: parasitic capacitances create poles, feedback networks create zeros, and the interplay determines bandwidth, gain, and stability.

The math is the same. The matrix is larger. But the principle -- solve ${tex`(G + j\omega C)\mathbf{x} = \mathbf{b}`} at each frequency -- doesn't change.
