# The Diode

The diode is the simplest nonlinear device in SPICE, and therefore the best place to understand how all nonlinear devices work.

A diode does one thing: it lets current flow easily in one direction and blocks it in the other. A silicon diode forward-biased at 0.65V might pass milliamps of current. Reverse the polarity and it passes essentially nothing — a few picoamps of leakage. This asymmetry, described by an exponential equation, is what makes the diode nonlinear and what makes it interesting for simulation.

Every technique introduced here — linearization, companion models, voltage limiting — applies directly to MOSFETs, BJTs, and every other semiconductor device in SPICE. The diode just makes them easier to see.

```js
import {kdlToSvg, SimBuilder, setWasmUrl} from "../components/spice.js";
setWasmUrl(FileAttachment("../wasm/spice_rs_wasm_bg.wasm").href);
import {Resistance, Capacitance, Voltage, Frequency, formatSpice, formatEE} from "../components/ee-inputs.js";
import {simPanel} from "../components/readout.js";

// Safe exponential — clamp argument to prevent overflow
function safeExp(x) { return Math.exp(Math.min(x, 80)); }
function diodeI(v, Is, nVt) { return Is * (safeExp(v / nVt) - 1); }
```

## The Shockley equation

The current through an ideal diode is:

```tex
I_D = I_s\left(e^{V_D / nV_t} - 1\right)
```

This is the **Shockley diode equation**, and it governs every PN junction in SPICE — not just standalone diodes, but also the junctions inside MOSFETs and BJTs.

### The parameters

**${tex`I_s`} — saturation current.** The tiny reverse-bias leakage current, typically around ${tex`10^{-14}`} A for a small silicon diode. It's small, but it sets the scale for the entire I-V curve. In the spice-rs `DiodeModel`, this is the `is` parameter (default: `1e-14`).

**${tex`n`} — emission coefficient (ideality factor).** A dimensionless number between 1 and 2. An ideal diode has ${tex`n = 1`}; real diodes have higher values because of recombination in the depletion region. Higher ${tex`n`} makes the exponential rise more gradual — the diode "turns on" at a slightly higher voltage.

**${tex`V_t`} — thermal voltage.** Defined as ${tex`V_t = kT/q`}, where ${tex`k`} is Boltzmann's constant, ${tex`T`} is temperature in Kelvin, and ${tex`q`} is the electron charge. At room temperature (300.15 K):

```tex
V_t = \frac{1.38065 \times 10^{-23} \cdot 300.15}{1.60218 \times 10^{-19}} \approx 0.02585 \text{ V} \approx 26 \text{ mV}
```

The thermal voltage is not a model parameter — it's a physical constant that depends only on temperature. At room temperature, every 26 mV increase in ${tex`V_D`} multiplies the current by ${tex`e \approx 2.72`}.

The product ${tex`nV_t`} appears so often that ngspice computes it once and calls it `vte`:

```rust
// From device/diode.rs
let vt = BOLTZMANN_OVER_Q * self.temp;   // kT/q
let vte = self.model.n * vt;             // n * kT/q
```

Three regimes are visible:

**Reverse bias** (${tex`V_D < 0`}): Current is approximately ${tex`-I_s`}. Essentially zero. The diode blocks.

**Below turn-on** (${tex`0 < V_D < 0.5`} V): Current is positive but negligibly small.

**Forward conduction** (${tex`V_D > 0.6`} V): The exponential takes off. A 50 mV change produces a 7x change in current. This extreme sensitivity is both what makes diodes useful (as switches and rectifiers) and what makes them challenging to simulate.

### Why the -1?

The equation has a ${tex`-1`} inside the parentheses: ${tex`I_s(e^{V/nV_t} - 1)`}. This ensures that at ${tex`V_D = 0`}, the current is exactly zero. It also means that in reverse bias, ${tex`I_D \to -I_s`} as ${tex`V_D \to -\infty`}. In practice, the ${tex`-1`} only matters when ${tex`V_D`} is very small or negative.

### The exponential challenge

The factor ${tex`V_D / nV_t`} in the exponent means that the argument to `exp()` grows at about 38.6 per volt (for ${tex`n = 1`}). At ${tex`V_D = 20`} V (which can easily happen during a bad NR iteration), the exponent is 772 — and ${tex`e^{772}`} is astronomically larger than what a 64-bit float can represent.

SPICE handles this with **voltage limiting** (below). But the exponential sensitivity is worth internalizing: it's the root cause of most diode convergence problems, and the same issue appears in every device with a PN junction.

## Linearization and companion models

At each Newton-Raphson iteration, every nonlinear device must be replaced by a linear equivalent that can be stamped into the MNA matrix. For the diode, this means turning the exponential I-V curve into a straight line — a **companion model** consisting of a conductance in parallel with a current source.

### The tangent-line approximation

Suppose we're at iteration ${tex`k`}, and the current guess for the diode voltage is ${tex`V_D^{(k)}`}. We evaluate the Shockley equation to get the current:

```tex
I_D^{(k)} = I_s\left(e^{V_D^{(k)} / nV_t} - 1\right)
```

and take its derivative to get the conductance (slope of the I-V curve at this point):

```tex
g_d^{(k)} = \frac{dI_D}{dV_D}\bigg|_{V_D^{(k)}} = \frac{I_s}{nV_t} \cdot e^{V_D^{(k)} / nV_t}
```

These two numbers define a tangent line to the I-V curve. It can be rearranged into a conductance in parallel with a current source:

```tex
I = g_d^{(k)} \cdot V + I_{eq}^{(k)} \qquad\text{where}\qquad I_{eq}^{(k)} = I_D^{(k)} - g_d^{(k)} \cdot V_D^{(k)}
```

The companion model stamps into the matrix exactly like a resistor plus a current source:

```tex
G[b,b] \mathrel{+}= g_d^{(k)} \qquad\qquad \text{RHS}[b] \mathrel{-}= I_{eq}^{(k)}
```

The tangent line crosses the true I-V curve at the operating point. To the left, it overestimates the current (the curve is concave up). To the right, it underestimates. This systematic error is what Newton-Raphson corrects by re-linearizing at each iteration.

### The companion circuit

The companion model is a Norton equivalent: a conductance ${tex`g_d`} in parallel with a current source ${tex`I_{eq}`}. In spice-rs, this is the stamping section of `device/diode.rs`:

```rust
// From device/diode.rs — load() function
let cdeq = cd - gd * vd;   // Norton equivalent current

// RHS stamps
mna.stamp_rhs(n, cdeq);    // positive at cathode
mna.stamp_rhs(pp, -cdeq);  // negative at anode

// Conductance stamps
mna.stamp(pp, pp, gd);
mna.stamp(n, n, gd);
mna.stamp(n, pp, -gd);
mna.stamp(pp, n, -gd);
```

### A critical detail: every device does this

The diode companion model is not specific to diodes. Every nonlinear device in SPICE follows exactly the same pattern: evaluate the device equations, compute the conductance (partial derivative), compute the Norton equivalent current source, and stamp both into the matrix.

For a MOSFET, the "conductances" include ${tex`g_m`} (transconductance), ${tex`g_{ds}`} (output conductance), and ${tex`g_{mbs}`} (body transconductance). More terminals and more partial derivatives, but the stamping mechanism is identical.

## Voltage limiting

Newton-Raphson proposes new voltages by solving a linearized system. The linearization is only accurate near the current operating point, so the proposed voltage might be far from the truth — and for a diode, "far from the truth" can mean catastrophe.

If the solver proposes ${tex`V_D = 20`} V, the exponent is 772 and `exp()` returns infinity. The simulation crashes.

**Voltage limiting** prevents this by clamping the proposed voltage change before the exponential is evaluated.

### The pnjlim algorithm

The limiting function in SPICE is called `pnjlim` (PN junction limit). It takes four inputs:
- `vnew` — the voltage proposed by the solver
- `vold` — the voltage from the previous iteration
- `vt` — the thermal voltage (${tex`nV_t`})
- `vcrit` — a critical voltage threshold

And returns a limited voltage that is safe to feed into `exp()`.

### The critical voltage

The threshold ${tex`V_{crit}`} is where limiting kicks in:

```tex
V_{crit} = nV_t \cdot \ln\!\left(\frac{nV_t}{\sqrt{2} \cdot I_s}\right)
```

For typical values (${tex`n = 1`}, ${tex`V_t = 0.02585`} V, ${tex`I_s = 10^{-14}`} A):

```tex
V_{crit} = 0.02585 \cdot \ln\!\left(\frac{0.02585}{\sqrt{2} \cdot 10^{-14}}\right) \approx 0.75 \text{ V}
```

In spice-rs, ${tex`V_{crit}`} is precomputed during temperature setup:

```rust
// From device/diode.rs — temperature()
self.t_vcrit = vte * (vte / (SQRT_2 * self.t_sat_cur)).ln();
```

### The limiting rules

**Case 1: Large positive voltage, big change.** If ${tex`V_{new} > V_{crit}`} and ${tex`|V_{new} - V_{old}| > 2V_t`}, the voltage change is compressed using a logarithm:

```tex
V_{limited} = V_{old} + V_t \cdot \left(2 + \ln\!\left(\frac{V_{new} - V_{old}}{V_t} - 2\right)\right)
```

A proposed jump of 5V becomes a step of maybe 0.1V. The solver's intent is preserved (increase the voltage) but the magnitude is tamed.

**Case 2: Negative voltage, big swing.** The voltage is clamped to prevent wild swings into deep reverse bias.

**Case 3: Normal region.** If the change is small or the voltage is below ${tex`V_{crit}`}, no limiting is applied.

### The check flag

When limiting is applied, `pnjlim` sets `check = true`. This flag propagates up to the NR loop as the `noncon` flag. Its meaning: "I had to override the solver's voltage, so the solution isn't self-consistent yet — don't declare convergence."

### Why not just use smaller steps?

Two reasons: (1) Only exponential devices need it — limiting resistors and capacitors would slow convergence for no benefit. (2) The limit depends on the device — a diode with ${tex`I_s = 10^{-14}`} has ${tex`V_{crit} \approx 0.75`} V; a Schottky with ${tex`I_s = 10^{-6}`} has ${tex`V_{crit} \approx 0.30`} V.

## Junction capacitance

For DC operating point analysis, the diode is fully described by its I-V curve. But for transient analysis (circuits with changing signals) and AC analysis (small-signal frequency response), the diode also stores charge. This charge storage shows up as **junction capacitance**.

### Depletion capacitance

When a diode is reverse-biased, the depletion region behaves like a parallel-plate capacitor. The SPICE model:

```tex
C_j = \frac{CJ_0}{\left(1 - V_D / VJ\right)^M}
```

where ${tex`CJ_0`} is the zero-bias junction capacitance (typically 1-100 pF), ${tex`VJ`} is the junction potential (~0.7-0.9 V for silicon), and ${tex`M`} is the grading coefficient (0.5 for an abrupt junction, 0.33 for linearly graded).

As ${tex`V_D`} approaches ${tex`VJ`}, the capacitance would go to infinity. SPICE avoids this singularity by switching to a linear extrapolation above the threshold ${tex`FC \cdot VJ`}.

The charge:

```tex
Q_{dep} = \frac{VJ \cdot CJ_0}{1 - M}\left[1 - \left(1 - \frac{V_D}{VJ}\right)^{1-M}\right]
```

### Diffusion capacitance

When forward-biased, minority carriers are injected across the junction. They take a finite time to recombine — the **transit time**, ${tex`TT`}. The stored charge is proportional to the forward current:

```tex
Q_{diff} = TT \cdot I_D
```

The corresponding capacitance is:

```tex
C_d = TT \cdot g_d
```

Diffusion capacitance dominates in forward bias — at a forward current of 10 mA with ${tex`TT = 10`} ns, the diffusion capacitance is about 4 nF, orders of magnitude larger than the typical depletion capacitance.

### Total junction capacitance

```tex
Q_{total} = Q_{dep} + Q_{diff} \qquad\qquad C_{total} = C_j + C_d
```

In reverse bias, ${tex`C_d \approx 0`}, so depletion capacitance dominates. In forward bias, ${tex`C_d`} grows exponentially and overwhelms ${tex`C_j`}.

In spice-rs, the capacitance adds more conductance and current to the companion model via the `ni_integrate` call, which converts ${tex`dQ/dt`} into an equivalent circuit element. For DC operating point, junction capacitance is irrelevant — nothing is changing, so ${tex`dQ/dt = 0`}.

The operating point is where the resistor's load line (${tex`I = (V_{in} - V_b) / R`}) intersects the diode's I-V curve.
```js
{
  const Is_l = 1e-14, nVt_l = 0.02585;

  const loadPts = Array.from({length: 100}, (_, i) => {
```

## Circuits: rectifiers

The diode's ability to conduct in only one direction makes it the core component of **rectifiers** — circuits that convert AC to DC.

### Half-wave rectifier

The simplest rectifier: a diode in series with the load.

```js
const hwKdl = `circuit "Half-Wave Rectifier" {
    group "main" {
        component "Vin" type="voltage-source" { value "AC 5V 60Hz"; port "pos" net="in"; port "neg" net="gnd"; place col=-3 row=0 }
        component "D1" type="diode" { model "DMOD"; port "anode" net="in"; port "cathode" net="out"; place col=0 row=0 }
        component "R1" type="resistor" { value "1k"; port "1" net="out"; port "2" net="gnd"; place col=0 row=2 }
        component "C1" type="capacitor" { value "100u"; port "1" net="out"; port "2" net="gnd"; place col=3 row=2 }
    }
    node "gnd" role="ground"
    node "out" label="Vout"
}`;
const hwSvg = await kdlToSvg(hwKdl);
const hwEl = document.createElement("div");
hwEl.className = "ferrite-circuit";
hwEl.innerHTML = hwSvg;
display(hwEl);
```

During the positive half-cycle, the input voltage rises above the output voltage plus the diode's forward drop (~0.65V). The diode conducts, charging the capacitor and delivering current to the load.

During the negative half-cycle, the input drops below the output. The diode is reverse-biased and blocks. The capacitor slowly discharges through the load resistor, holding the output near its peak value.

The result is a DC output with **ripple** — small periodic dips as the capacitor discharges between conduction pulses.

Here is what the half-wave rectifier waveforms look like:

```js
{
  const Vpk = 5, f = 60, R = 1000, C = 100e-6;
  const dt = 1e-5, T = 50e-3;
  const pts_in = [], pts_out = [];
  let vout = 0;
  for (let t = 0; t < T; t += dt) {
    const vin = Vpk * Math.sin(2 * Math.PI * f * t);
    // Simple diode model: conducts when vin > vout + 0.65
    if (vin > vout + 0.65) {
      // Diode conducts — capacitor charges toward vin - 0.65
      const tau_charge = 10 * dt; // fast charge through diode Rs
      vout += (vin - 0.65 - vout) * (1 - Math.exp(-dt / tau_charge));
    } else {
      // Diode off — capacitor discharges through R
      vout *= Math.exp(-dt / (R * C));
    }
    if (t % (4e-4) < dt) {
      pts_in.push({t: t * 1000, v: vin, signal: "V_in"});
      pts_out.push({t: t * 1000, v: vout, signal: "V_out"});
    }
  }
  display(Plot.plot({
    width: 640,
    height: 300,
    marginLeft: 60,
    style: {fontSize: "13px"},
    x: {label: "Time (ms)"},
    y: {label: "Voltage (V)", domain: [-6, 6]},
    color: {legend: true, domain: ["V_in", "V_out"], range: ["#888", "#b87333"]},
    marks: [
      Plot.ruleY([0], {stroke: "#ccc"}),
      Plot.line(pts_in, {x: "t", y: "v", stroke: "signal", strokeWidth: 1}),
      Plot.line(pts_out, {x: "t", y: "v", stroke: "signal", strokeWidth: 2})
    ]
  }));
}
```

The diode "costs" about 0.65V: the peak output is about 4.35V instead of 5V. With a real diode model that includes series resistance (RS), the drop is slightly higher under load.

### Full-wave bridge rectifier

A bridge rectifier uses four diodes to capture both half-cycles:


On the positive half-cycle (${tex`V_{in+} > V_{in-}`}), diodes D1 and D4 conduct. On the negative half-cycle, D3 and D2 conduct. The load sees a full-wave rectified signal — twice the frequency, half the ripple compared to the half-wave rectifier.

Two diode drops are in the path (about 1.3V total), so the peak output from a 12V peak source is about 10.7V.

```js
{
  const Vpk = 12, f = 60, R = 1000, C = 470e-6;
  const dt = 1e-5, T = 100e-3;
  const pts_in2 = [], pts_out2 = [];
  let vout2 = 0;
  for (let t = 0; t < T; t += dt) {
    const vin = Vpk * Math.sin(2 * Math.PI * f * t);
```

### What to observe

When studying these simulations, look for:

- **Ripple voltage** — the periodic dip in ${tex`V_{out}`} between conduction pulses. Larger capacitance means less ripple.
- **Diode current pulses** — the diode conducts only near the peaks of the input waveform, in brief, high-current pulses. The peak diode current is much higher than the DC load current.
- **Forward voltage drop** — the ~0.65V offset between input peak and output peak. It increases slightly with current due to series resistance.
- **Reverse recovery** — if the diode model includes transit time (TT), there's a brief moment when the diode continues conducting after the voltage reverses, as the stored minority-carrier charge is swept out.

Bridge rectifiers are interesting for simulation because they have rapid diode switching events. Twice per cycle, two diodes turn off and two turn on. At each switching event, the NR iteration must track the transition from forward conduction to reverse blocking. The series resistance RS creates an internal node inside each diode, which adds unknowns to the matrix but improves convergence by limiting how quickly the junction voltage can change.

### SPICE Rectifier Simulation

The waveforms above use a simplified JavaScript model of the diode. Now let's run the **actual SPICE simulator** on a half-wave rectifier with a filter capacitor. Every diode evaluation uses the full Shockley equation, Newton-Raphson converges the nonlinear system at each timestep, and the trapezoidal rule integrates the capacitor charge.

```js
{
  const times = rectResult.times;
  const vin = rectResult.signals["v(vin)"];
  const vout = rectResult.signals["v(vout)"];

  // Build combined dataset for both traces
  const plotData = [];
  for (let i = 0; i < times.length; i++) {
    plotData.push({t: times[i] * 1000, v: vin[i], signal: "V(vin)"});
    plotData.push({t: times[i] * 1000, v: vout[i], signal: "V(vout)"});
  }

  // Find the last positive-going zero crossing of Vin to mark the last cycle
  const period = 1 / freq;
  const tEnd = times[times.length - 1];
  let lastZeroCross = tEnd - period; // fallback
  for (let i = times.length - 2; i > 0; i--) {
    if (vin[i] <= 0 && vin[i + 1] > 0 && times[i] < tEnd - period * 0.5) {
      lastZeroCross = times[i];
      break;
    }
  }
  // Compute ripple: max - min of vout in that last cycle
  let vMax = -Infinity, vMin = Infinity;
  for (let i = 0; i < times.length; i++) {
    if (times[i] >= lastZeroCross && times[i] <= lastZeroCross + period) {
      if (vout[i] > vMax) vMax = vout[i];
      if (vout[i] < vMin) vMin = vout[i];
    }
  }
  const ripple = vMax - vMin;
  const lastCycleStart = lastZeroCross;

  display(Plot.plot({
    width: 640,
    height: 320,
    marginLeft: 60,
    style: {fontSize: "13px"},
    x: {label: "Time (ms)"},
    y: {label: "Voltage (V)", grid: true},
    color: {legend: true, domain: ["V(vin)", "V(vout)"], range: ["#888", "#b87333"]},
    marks: [
      Plot.ruleY([0], {stroke: "#ccc"}),
      Plot.line(plotData, {x: "t", y: "v", stroke: "signal", strokeWidth: d => d.signal === "V(vout)" ? 2 : 1}),
      // Ripple annotation band in the last cycle
      Plot.rect([{x1: lastCycleStart * 1000, x2: tEnd * 1000, y1: vMin, y2: vMax}], {
        x1: "x1", x2: "x2", y1: "y1", y2: "y2",
        fill: "#b87333", fillOpacity: 0.1, stroke: "#b87333", strokeDasharray: "3,2", strokeWidth: 0.8
      }),
      Plot.text([{x: (lastCycleStart + period) * 1000, y: vMax}], {
        x: "x", y: "y",
        text: d => `ripple: ${formatEE(ripple, "V")}`,
        dy: -8, dx: -4, textAnchor: "end", fill: "#b87333", fontSize: 11, fontWeight: 600
      })
    ]
  }));

  display(html`<p style="font-size:0.85em;color:#6b5d4d;font-style:italic;">
    Peak output: ${vMax.toFixed(3)}V (diode drop reduces ${vpeak}V peak input by ~0.65V).
    Ripple voltage: <strong>${(ripple * 1000).toFixed(1)} mV</strong> — increase C or R to reduce it.
    ${rectResult.accepted != null ? `${rectResult.accepted} accepted timesteps, ${rectResult.rejected} rejected.` : ""}
  </p>`);
}
```

The SPICE result captures effects the simplified model misses: the exact forward drop depends on the current (not a fixed 0.65V), the capacitor charges through the diode's dynamic resistance, and the simulator's adaptive timestep concentrates effort around the conduction pulses where the diode switches rapidly between forward and reverse bias.

Try increasing the capacitance — the ripple drops because the RC discharge time constant grows. Increasing the load resistance has the same effect (less current drawn from the capacitor between conduction pulses). Increasing frequency also reduces ripple because the capacitor has less time to discharge between peaks.
